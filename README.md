# Experimental ImmortalWrt Builds for mt798x

This repository builds custom firmware images based on
[SuperKali/immortalwrt-mt798x-rebase](https://github.com/SuperKali/immortalwrt-mt798x-rebase),
branch `25.12-linkup`.

The GitHub Actions workflow clones that upstream tree, applies local patches from
`patches/`, copies the rootfs overlay from `files/`, merges the baseline and
device-specific defconfigs, and then builds one image per device profile.

## Current Targets

- **Zyxel EX5601-T0 (ubootmod)**
- **GL.iNet GL-MT3000 / Beryl AX**

## Build Behavior

- Builds run only on commits pushed to `main` and tags matching `v*`.
- Scheduled upstream-change checks are disabled; upstream changes do not trigger builds by themselves.
- Successful push/tag builds publish zipped firmware artifacts as draft GitHub Releases.

## Default Configuration

The image is intended to boot preconfigured as a dumb access point.

Network defaults from `files/etc/uci-defaults/99-ap-mode`:

- LAN is set to static `10.0.0.9/24`.
- DHCPv4 is disabled on LAN.
- DHCPv6, Router Advertisements, and NDP are disabled on LAN.
- LAN IPv6 prefix assignment is removed.
- The AP itself uses `10.0.0.1` as its default gateway and DNS server.

Wi-Fi defaults from `patches/0005-mtwifi-cfg-ucode-change-default-wifi-settings.patch`:

- Wi-Fi country defaults to `SE`.
- 2.4 GHz defaults to channel `6`, `HE40`, SSID `ImmortalWrt-2.4G`.
- 5 GHz defaults to channel `auto`, `HE80`, SSID `ImmortalWrt-5G`.
- 6 GHz defaults to channel `auto`, `HE160`, SSID `ImmortalWrt-6G`.
- Default wireless mode is AP on the `lan` network.
- Default encryption is WPA2-PSK with key `password123`.

Other local changes:

- Retains MediaTek Wi-Fi support from the upstream build while excluding many VPN-related packages via defconfig.
- Sets LuCI/default language choices toward English.
- Adds/fixes MediaTek Wi-Fi region and DFS-related behavior through local patches.

## Installation

Compiled builds are published as draft GitHub Releases for people with write
access to this repository:

1. Open the draft release generated by the workflow.
2. Download the zipped firmware asset for your device.
3. Extract the zip.
4. Flash the `*squashfs-sysupgrade.itb` file to a supported device already running OpenWrt/ImmortalWrt ubootmod.

After first boot, connect through the upstream network and manage the AP at
`10.0.0.9`. The upstream router is expected to be reachable at `10.0.0.1`.
